NPM简介

首先介绍Node.js：

Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行环境。Node.js 使用了一个事件驱动、非阻塞式 I/O 的模型，使其轻量又高效。简单可以理解为javascript可以作为一门服务器端语言来进行开发。

为什么要使用Node.js

nodejs在服务端与其他语言相比有个很大的优势就是非阻塞IO，专为网络服务而设计，高效的解决输入和输出，nodejs使用事件循环和线程池的方式来解决高并发的问题，但是对于大量的计算，nodejs却并不适合。

Node.js的安装和版本管理

  • 最简单的安装方式，请上官网下载安装包，双击安装一路next，这里不做介绍。
  • 使用NVM来安装和管理（推荐使用）
    1.下载NVM包，请确保你的网络通畅。
    2.选择第一个：nvm-noinstall.zip 下载完成后解压到一个地方，比如解压到C:\dev\nvm。
    3.双击 install.cmd 然后会让你输入“压缩文件解压或拷贝到的一个绝对路径” 先不用管它，直接回车，成功后，会在C盘的根目录生成一个settings.txt的文本文件，把这个文件剪切到C:\dev\nvm目录中，然后我们把它的内容修改成这样：

    root: C:\dev\nvm
    path: C:\dev\nodejs
    arch: 64
    proxy: none
    node_mirror: http://npm.taobao.org/mirrors/node/
    npm_mirror: https://npm.taobao.org/mirrors/npm/

    5.然后我们开始配置环境变量了，因为刚刚点击了install.cmd的文件，那么会在环境变量的系统变量中，生成两个环境变量：NVM_HOME 和NVM_SYMLINK 我们开始修改这两个变量名的变量值：NVM_HOME的变量值为：C:\dev\nvm； NVM_SYMLINK的变量值为：C:\dev\nodejs
    6.在用户环境变量里输入： ;%NVM_HOME%;%NVM_SYMLINK%;
    7.打开一个cmd窗口输入命令：nvm v ，那么我们会看到当前nvm的版本信息。然后我们可以安装nodejs了。
    8.继续输入命令：nvm install latest 我们会看到正在下载的提示，下载完成后 会让你use那个最新的node版本。
    9.如果你是第一次下载，在use之前，C:\dev目录下是没有nodejs这个文件夹的，在输入比如： nvm use 6.2.0 之后，你会发现，C:\dev目录下多了一个nodejs文件夹，这个文件夹不是单纯的文件夹，它是一个快捷方式，指向了 C:\dev\nvm 里的 v6.2.0 文件夹。
    10.同样的咱们可以下载其他版本的nodejs，这样通过命令:nvm use 版本号 比如：nvm use 5.11.0就可以轻松实现版本切换了。你可以看到nodejs快捷方式的指向已经改成当前版本的了。
    11.备注： 如果你的电脑系统是32 位的，那么在下载nodejs版本的时候，一定要指明 32 如： nvm install 6.2.0 32 这样在32位的电脑系统中，才可以使用，默认是64位的。

NPM的介绍

  • NPM是Node.js的包管理器，是目前全球最大的开源库生态系统。
  • 首先我们进入命令模式，输入 npm config set prefix “C:\dev\nvm\npm”回车，这是在配置npm的全局安装路径，然后在用户文件夹下会生成一个.npmrc的文件。
  • 接着请在下面添加一栏：registry=https://registry.npm.taobao.org 它是为了绑定淘宝镜像的 因为有时候国外的容易被墙
  • 然后继续在命令中输入： npm install npm -g 回车后会发现正在下载npm包，在C:\dev\nvm\npm目录中可以看到下载中的文件，以后我们只要用npm安装包的时候加上 -g 就可以把包安装在我们刚刚配置的全局路径下了。如果不加-g 就是安装在当前目录文件夹内。
  • 我们为这个npm配置环境变量： 变量名为：NPM_HOME，变量值为 ：C:\dev\nvm\npm
  • 在用户环境变量的最前面添加;%NPM_HOME%，注意了，这个一定要添加在 %NVM_SYMLINK%之前，所以我们直接把它放到用户环境变量的最前面。
  • 最后我们新打开一个命令窗口，输入npm -v ,此时我们使用的就是我们统一下载的npm包了。
  • 如果你觉得还是挺慢的 同样的我们还可以安装cnpm工具，它是中国版的npm镜像库，地址在这里：https://cnpmjs.org/，也是npm官方的一个拷贝，因为我们和外界有一堵墙隔着，所以用这个国内的比较快，淘宝也弄了一个和npm一样的镜像库，http://npm.taobao.org/，它和官方的npm每隔10分钟同步一次。安装方式：
    1.npm install -g cnpm –registry=http://r.cnpmjs.org
    2.或者用淘宝的npm install -g cnpm –registry=https://registry.npm.taoba.org
    3.安装好了cnpm后，直接执行cnpm install 包名比如：cnpm install bower -g 就可以了。-g只是为了把包安装在全局路径下。如果不全局安装，也可以在当前目录中安装，不用-g就可以了。

前端项目的基本结构

  • 初始化一个空的前端目录，使用npm init 命令，然后一路next(可以使用指令 npm init -y 全部默认)，会生成一个package.json的配置文件。
  • 配置文件中需要关注的三个设置项：
    1.script:可以配置一些Node环境下直接执行的指令，然后使用指令: npm run [key]来执行一些比较复杂的指令，其实类似于指令的快捷方式。
    2.dependencies:项目生产环境的依赖项(最终上线时项目以来的包，比如jQuery)，通常使用:npm install [package] --save指令安装的依赖包都会将包名和版本号配置在此选项中。
    3.devDependencies:项目开发环境的依赖项，通常使用:npm install [package] --save-dev指令安装的依赖包都会将包名和版本号配置在此选项中。
"script": {
    "dev": "webpack --config webpack.conf.js"
},
"dependencies": {
  "element-ui": "^1.3.4",
  "vue": "^2.2.6",
  "vue-resource": "^1.3.3",
  "vue-router": "^2.3.1"
},
"devDependencies": {
   "autoprefixer": "^6.7.2",
   "babel-core": "^6.22.1",
}
  • 初始化为git仓库，使用git init命令，然后创建.gitignore文件，来排除git版本管理的文件或目录，具体配置自行百度之后按需配置

Vue项目的基本搭建

  • 按官网推荐的方式:

    //全局安装 vue-cli
    npm install –global vue-cli
    //创建一个基于 webpack 模板的新项目
    vue init webpack my-project
    //安装依赖，走你
    cd my-project
    npm install
    npm run dev

  • 运行起来初始化的项目之后，修改组件的内容可以看到页面会自动刷新，这用到了webpack的热更新，后续的webpack教程会提及到。
